# 🚀 GemGPT 快速开始 - 实时预览开发

## 一键启动实时预览

### 方法1: 使用启动脚本（最简单）
```bash
./start-dev.sh
```

### 方法2: 使用 npm 命令
```bash
npm run dev
```

## 📐 开发工作流

### 步骤1: 启动开发服务器
在终端运行：
```bash
npm run dev
```

### 步骤2: 浏览器自动打开
- 开发服务器会在 `http://localhost:3000` 自动打开浏览器
- 如果没有自动打开，手动访问: `http://localhost:3000`

### 步骤3: 开始开发
- **左侧**: 在 Cursor 中编辑代码
- **右侧**: 浏览器实时显示更改

## ✨ 实时预览特性

### Hot Module Replacement (HMR)
- ✅ 修改 React 组件 → 自动更新（保持状态）
- ✅ 修改 CSS 样式 → 立即生效
- ✅ 修改 TypeScript → 自动重新编译
- ✅ 无需手动刷新页面

### 错误提示
- 🔴 编译错误 → 显示在浏览器中
- 🔴 运行时错误 → 显示在浏览器控制台
- ⚠️ TypeScript 错误 → 显示在终端

## 🛠️ 开发命令

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动开发服务器（自动打开浏览器） |
| `npm run dev:host` | 启动开发服务器（允许外部访问，用于手机预览） |
| `npm run build` | 构建生产版本 |
| `npm run preview` | 预览生产构建 |

## 📱 移动端预览

### 使用浏览器 DevTools
1. 打开浏览器 DevTools (F12)
2. 点击设备工具栏图标（或 `Cmd+Shift+M` / `Ctrl+Shift+M`）
3. 选择设备（iPhone, iPad, Android等）
4. 查看响应式效果

### 在真实手机上预览
1. 运行: `npm run dev:host`
2. 查看终端输出的网络地址（如: `http://192.168.1.100:3000`）
3. 在手机浏览器中输入该地址

## 🐛 常见问题

### 端口被占用
修改 `vite.config.ts` 中的 `server.port` 为其他端口（如 3001）

### 更改不显示
- 硬刷新浏览器: `Cmd+Shift+R` (Mac) 或 `Ctrl+Shift+R` (Windows)
- 清除浏览器缓存
- 重启开发服务器

### HMR 不工作
- 检查文件是否在 `src/` 目录下
- 检查是否有语法错误
- 尝试重启开发服务器

## 💡 开发技巧

1. **保持浏览器打开**: 开发时始终在浏览器中查看效果
2. **使用浏览器 DevTools**: 检查元素、查看控制台、监控网络
3. **及时测试**: 每完成一个小功能就测试
4. **检查控制台**: 及时查看错误和警告

---

**提示**: 开发时保持 `npm run dev` 运行，这样每次保存文件都会自动更新浏览器预览！
